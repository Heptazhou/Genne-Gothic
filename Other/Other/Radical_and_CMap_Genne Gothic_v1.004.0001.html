Version 1.004

前言

修改思源黑體的目的，是用於實現思源黑體舊字形的顯示、文書處理、圖形處理及嵌入至行動裝置等應用。

由於ADOBE原先提供的思源黑體只有韓國版跟舊字形相似，並且因特徵（Features）裡的查找（Lookup）表內，查找39~41'locl'本地化 形式有指定部分語言下顯示的字形（Glyph），導致在指定為繁中或簡中的環境都顯示為臺標或陸標，所以本人對字型的CMap以及GSUB做了些修改，以盡量 在繁簡體環境中讓字型顯示為舊字形。

所需之應用軟體

unofficial fontforge-cygwin

字型編輯軟體，雖然也可以修改Glyph對應，但由於無法正常開啟思源黑體（當某Glyph對應多個映射時第二個以後的映射會遺失），且儲存時會遇到種種問 題（直排出錯，Table遺失等等）， 所以在這裡不用於修改字型。
在這裡用作CID查詢工具，使用時記得移除“fontforge-cygwin”資料夾底下“\cygwin\usr\share\fontforge”裡 所有附檔名(後輟)為.cidmap的檔案，否則會按照cidmap檔案映射字圖而導致映射錯亂。
解壓縮至任一路徑並執行“fontforge.bat”內即可使用。

Adobe Font Development Kit for OpenType(AFDKO)

編譯思源黑體會用到的編譯工具。
解壓縮後放置至任意路徑，然後以系統管理員的身分執行FDK資料夾底下的“FinishInstallWindows.cmd”安裝編譯環境（之後請不要任意 變更路徑，若變更了路徑，需再重新執行“FinishInstallWindows.cmd”一次）。

思源黑體 1.004或更高版本：

1.004 源代碼

源 代碼搭配AFDKO生 成字型檔用，請下載1.004 源代碼、Resources內 的“utf32-kr.map”CMap 檔案以及思 源黑體任一版任一字重（任一國家版本的CID字圖排列方式都是相同的，僅映射不同），並將1.004 源代碼放置至任意路徑以方便進行接下來的作業。

製作流程

參考：原创 用Adobe FDK生成自定义思源OTF（更新旧字形1.001版） - 极限社区

1.建構開發者環境

建構開發者環境並下載源代碼包，請參考上方“所需之應用軟體”章節的內容。

2.修改CMap文件

2-1.前置作業

新建一個資料夾以方便處理CMap檔案。

將source-han-sans-1.004資 料夾下的“UniSourceHanSansKR-UTF32-H”、“utf32-kr.map”， 以及“cmap-tool.pl” 腳本複製到新建的資料夾內。

2-2.建立用於處理CMap的檔案

將“UniSourceHanSansKR-UTF32-H”重新命名為“temp”。

用文字編輯軟體開啟“temp”，並將第一個“100 begincidchar”與最後一個“endcidrange”之間的數值全部刪除。

然後用文字編輯軟體開啟“utf32-kr.map”， 並將內容全選複製，貼上到到temp檔案內的第一個“100 begincidchar”與最後一個“endcidrange”之間後存檔。

2-3.檢索並修改映射

2-3-1.使用FontForge找 出Glyph的CID對應

開啟FontForge，然後開啟 思源黑體。

開啟之後點選「選單列→CID→平面化」。

2-3-1-1.用Unicode檢索

使用「選單列→檢視→前往」來選取Glyph並查看Glyph名稱。

例如想要找到對應「月」的Glyph，點選「選單列→檢視→前往」然後在輸入欄位內輸入“U+6708”後按下確定，此時主視窗的選擇游標將會移動到「月」 的Glyph上。

2-3-1-2.用Glyph名稱檢索

由於Glyph名稱與CID相同，所以可以用此方式來找出Glyph。

與上方舉例相同，若要用CID的方式找到「月」的Glyph，只要將輸入欄位內改為“Identity.21090”即可。

2-3-2.得知Glyph的CID值

點選欲查詢的Glyph，此時選單列下方會顯示一段文字。

例如點選「月」的Glyph，文字會顯示“21090 (0×5262) U+6708 “Identity.21090” CJK Unified Ideographs”，這裡的“Identity.21090”內的“21090”就是「月」目前的CID。

2-3-3.修改CID映射

開啟剛剛建立的temp檔案。

例如我要修改「月」字的映射，映射到Glyph名為“Identity.62270”的Glyph：

將

<00006708>    21090

取代為

<00006708>    62270

後存檔即可。

2-4.生成新的CMap檔案

在存放“temp”的資料夾下開啟cmd（按住Shift→空白處右鍵→在此處開啟命令視窗），鍵入命令：

perl cmap-tool.pl < temp > UniSourceHanSansKR-UTF32-H

以生成“UniSourceHanSansKR-UTF32-H”CMap檔案。

3.生成自訂的思源黑體

將“UniSourceHanSansKR-UTF32-H”放到source-han-sans-1.004資 料夾底下。

修改字型名稱

用文字編輯軟體開啟FontMenuNameDB。


[SourceHanSans-Regular]
    f=3,1,0x411,\6E90\30CE\89D2\30B4\30B7\30C3\30AF
    s=3,1,0x411,Regular
    l=3,1,0x411,\6E90\30CE\89D2\30B4\30B7\30C3\30AF Regular
    f=Source Han Sans
    s=Regular
    l=Source Han Sans Regular


說明：

[SourceHanSans-[字重]]
    f=3,1,[語系ID(Hex)],[字型家族]
    s=3,1,[語系ID(Hex)],[字重]
    l=3,1,[語系ID(Hex)],[字型名稱]
    f=[英文-字型家族]
    s=[英文-字重]
    l=[英文-字型名稱]


後續選擇的CIDFonts是決定選擇哪個SourceHanSans標籤的關鍵，下列為標籤的關係：

標籤 CIDFonts
[SourceHanSans-[字重]] cidfont.ps.OTC.J
 [SourceHanSansK-[字重]] cidfont.ps.OTC.K
[SourceHanSansSC-[字重]] cidfont.ps.OTC.SC
[SourceHanSansTC-[字重]] cidfont.ps.OTC.TC


標籤 名稱
f
 字型家族
s
 字重
l
 字型名稱

以下為語系ID的Hex值：

語言 語系ID(Hex)
日文 0x411
韓文 0x412
簡體中文 0x804
繁體中文 0x404

文字編碼

文字編碼爲CSS，結構爲反斜線加上Unicode碼。
可用Unicode 轉換工具轉換爲正字。


建立腳本

在source-han-sans-1.004資 料夾下建立一個名為“output.bat”的腳本，然後用文字編輯軟體開啟。


範例：


makeotf -f cidfont.ps.OTC.TC -ff features.OTC.TC -fi cidfontinfo.OTC.TC -mf ../../FontMenuNameDB -r -nS -cs 1 -ch ../../UniSourceHanSansKR-UTF32-H -ci ../../SourceHanSans_KR_sequences.txt


說明：


makeotf -f [CIDFonts] -ff [Features] -fi [CIDFontInfo] -mf ../../[FontMenuNameDB] -r -nS -cs 1 -ch ../../[CMap] -ci ../../[Sequences]



CIDFonts

沒有套用任何Glyph映射的CID字型檔。


Features

特徵（Features），就是控制直排時切換合適的標點符號，不同語言環境下文字套用不同映射等等的查找（LookUp）清單。

如果是繁體中文使用者，請填features.OTC.TC，
如果是簡體中文使用者，請填features.OTC.SC；

選取的語系會影響到顯示，假如你是繁體中文使用者，但選了簡中，字型嵌入到安卓裝置上或套用在電腦網頁等等，並且網頁有指定語言的情況下會發生原先套用陸標字但顯 示爲台標字的現象，反之亦然。

如果要移除'locl'本地化形式請使用ttx工具修改GSUB Table。


CIDFontInfo

字型資訊，版本號等等。

如果是繁體中文使用者，請填cidfontinfo.OTC.TC，
如果是簡體中文使用者，請填cidfontinfo.OTC.SC。


FontMenuNameDB

字型名稱，會因為CIDFonts的數值來決定要套用何種語言的字型名稱。


CMap

字元映射表。


Sequences

變體選擇器，IVS（ideographic variation sequence; 異体字シーケンス），不包含此資訊也可生成字型。


批次檔腳本範例

cd ExtraLight/OTC
call makeotf -f cidfont.ps.OTC.TC -ff features.OTC.TC -fi cidfontinfo.OTC.TC -mf ../../FontMenuNameDB -r -nS -cs 1 -ch ../../UniSourceHanSansKR-UTF32-H
cd ../../
cd Light/OTC
call makeotf -f cidfont.ps.OTC.TC -ff features.OTC.TC -fi cidfontinfo.OTC.TC -mf ../../FontMenuNameDB -r -nS -cs 1 -ch ../../UniSourceHanSansKR-UTF32-H
cd ../../
cd Normal/OTC
call makeotf -f cidfont.ps.OTC.TC -ff features.OTC.TC -fi cidfontinfo.OTC.TC -mf ../../FontMenuNameDB -r -nS -cs 1 -ch ../../UniSourceHanSansKR-UTF32-H
cd ../../
cd Regular/OTC
call makeotf -f cidfont.ps.OTC.TC -ff features.OTC.TC -fi cidfontinfo.OTC.TC -mf ../../FontMenuNameDB -r -nS -cs 1 -ch ../../UniSourceHanSansKR-UTF32-H
cd ../../
cd Medium/OTC
call makeotf -f cidfont.ps.OTC.TC -ff features.OTC.TC -fi cidfontinfo.OTC.TC -mf ../../FontMenuNameDB -r -nS -cs 1 -ch ../../UniSourceHanSansKR-UTF32-H
cd ../../
cd Bold/OTC
call makeotf -f cidfont.ps.OTC.TC -ff features.OTC.TC -fi cidfontinfo.OTC.TC -mf ../../FontMenuNameDB -r -nS -cs 1 -ch ../../UniSourceHanSansKR-UTF32-H
cd ../../
cd Heavy/OTC
call makeotf -f cidfont.ps.OTC.TC -ff features.OTC.TC -fi cidfontinfo.OTC.TC -mf ../../FontMenuNameDB -r -nS -cs 1 -ch ../../UniSourceHanSansKR-UTF32-H
cd ../../
move ExtraLight\OTC\SourceHanSans-ExtraLight.otf .
move Light\OTC\SourceHanSans-Light.otf .
move Normal\OTC\SourceHanSans-Normal.otf .
move Regular\OTC\SourceHanSans-Regular.otf .
move Medium\OTC\SourceHanSans-Medium.otf .
move Bold\OTC\SourceHanSans-Bold.otf .
move Heavy\OTC\SourceHanSans-Heavy.otf .
rename SourceHanSans-ExtraLight.otf SourceHanSansOldFormTC-ExtraLight.otf
rename SourceHanSans-Light.otf SourceHanSansOldFormTC-Light.otf
rename SourceHanSans-Normal.otf SourceHanSansOldFormTC-Normal.otf
rename SourceHanSans-Regular.otf SourceHanSansOldFormTC-Regular.otf
rename SourceHanSans-Medium.otf SourceHanSansOldFormTC-Medium.otf
rename SourceHanSans-Bold.otf SourceHanSansOldFormTC-Bold.otf
rename SourceHanSans-Heavy.otf SourceHanSansOldFormTC-Heavy.otf
pause

存檔後並執行指令就可以獲得7種字重的自訂字型了。
